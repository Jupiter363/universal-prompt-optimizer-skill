# Changelog

## [0.2.0] - 2026-05-23

### Added
- **追问模式**：缺失关键信息时主动向用户提问澄清，而非被动列出"缺失信息"
- 追问优先级系统：安全 > 目标 > 范围 > 技术细节 > 输出格式
- **信息熵止步机制**：方向性维度明确后立即停止追问，不等完美信息
- 追问节律控制：每轮 1-3 问，硬上限 3 轮，接受"不知道"回答
- 追问结束后的输出格式整合与追问汇总
- **提示词标准化（阶段二）**：追问结束后，按 LLM 偏好的 7 段结构（角色/任务/背景/目标/约束/步骤/输出格式）重组 Prompt

### Changed
- 核心原则重写：新增"先追问，不编造"、"用信息熵判断停止时机"、"第二阶段必须标准化"
- 执行流程重构为两阶段：追问澄清 → 提示词标准化
- 追问结束信号重新排序：信息熵达标 > 用户主动结束 > 3 轮上限 > 连续不知道
- 输出格式重构：元信息（追问汇总）+ 标准化 Prompt 分离
- 常见错误表扩展，新增追问和信息熵相关反模式
- 安全边界新增追问相关规则
- **默认行为改为直接执行**：标准化后不确认，直接执行任务。用户说"只优化不执行"才跳过执行
